# 2026-02-19

## 00:20 밤 — 트랜스크립트 타임스탬프 정렬 개선

### 무엇을 했는가
- 에피소드 페이지 트랜스크립트의 타임스탬프(hanging indent) 레이아웃이 1시간 이상 에피소드에서 정렬이 깨지는 버그를 수정함
- 3단계에 걸쳐 반복 개선

1. **1차 수정**: `min-width: 3.25rem` → `width: 5rem` 고정, `padding-left: 4rem` → `6rem`, `margin-left: -4rem` → `-6rem`, `margin-right: 0.5rem` → `1rem`
2. **2차 수정**: 여백 과다·중앙정렬 문제 개선 — `padding-left: 6rem` → `5rem`, `text-align: center` → `right`, `margin-right: 1rem` → `0`, `padding: 0.125rem 0.375rem` → `0.125rem 0.75rem 0.125rem 0` (padding-right를 gap으로 활용)
3. **3차 수정**: 첫 줄 미세 들여쓰기(~1/4글자) 수정 — `margin-right: 0` → `-0.3rem` (공백 문자 상쇄)

### 무엇이 잘 됐고, 무엇이 안 됐는가

**잘 된 것**
- 원인 분석이 정확했음: min-width 가변 → 고정 width, 공백 문자 문제 등

| 문제 | 원인 | 해결 |
|---|---|---|
| 1:00:00 이상 타임스탬프 시 텍스트 밀림 | `min-width: 3.25rem` 초과 시 박스가 커져 텍스트 밀림 | 고정 `width: 5rem`으로 변경 |
| 여백이 너무 넓어짐 | padding-left를 4→6rem으로 과도하게 확대 | 5rem으로 축소 + padding-right를 gap으로 활용 |
| 타임스탬프 중앙정렬 | `text-align: center` | `text-align: right`로 변경 (가상 세로선 기준 우측 정렬) |
| 첫 줄만 ~1/4글자 들여쓰기 | MDX 소스 `<span>` 뒤 공백 문자가 인라인 흐름에서 ~4.5px 렌더링 | `margin-right: -0.3rem`으로 상쇄 |

**알게 된 것 / gotchas**
- 개발 서버(`npm run dev`)에서 Tailwind v4 + `@tailwindcss/vite` 조합 시 스타일이 적용 안 된 것처럼 보이는 이슈 있음 (실제 프로덕션 빌드는 정상). 브라우저 강력 새로고침 또는 `.astro` 캐시 삭제 후 재시작으로 해결 시도했으나 dev 환경에서는 재현 안 됨 → 알려진 Tailwind v4 dev 이슈로 추정
- MDX에서 `<span>` 바로 뒤 공백이 인라인 `inline-block` 요소와 조합될 때 첫 줄에만 공백이 렌더링되어 정렬 불일치 발생. CSS `margin-right` 음수값으로 상쇄하는 패턴 유효

### 핵심 결정 및 근거

| 결정 | 근거 |
|---|---|
| `min-width` → 고정 `width` | 가변 너비는 긴 타임스탬프가 들어올 때 텍스트 시작 위치가 달라지는 근본 원인 |
| `padding-right`를 gap으로 활용, `margin-right: 0` | gap이 박스 내부에 있으면 텍스트 흐름에서 일관된 위치 보장 |
| `text-align: right` | 가상의 세로 기준선 오른쪽에 타임스탬프가 붙는 시각적 의도 구현 |
| `margin-right: -0.3rem` | MDX 소스 수정 없이 CSS만으로 공백 문자 렌더링 상쇄 |
