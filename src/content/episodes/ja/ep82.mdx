<ResourceLink url="https://x.com/METR_Evals/status/2002203627377574113" title="x.com" domain="x.com" />

<span class="paragraph-timestamp" data-ts="01:34">01:34</span> **チェ・スンジュン** まずニュース的な話題をいくつか触れると、2025年12月20日にこのグラフが更新されていたのですが、たぶん紹介し損ねていました。これは何度か紹介したMETRの50%-time horizonに関するものです。線形スケールで見ると、Claude Opus 4.5が4時間49分、ほぼ5時間まで大きく跳ね上がっているのが分かります。かなり前進しています。logスケールで見ると緩やかな伸びに見えるんですが、こちらで見るとこういう感じです。

ここで面白いのはタイトルです。Andrejと、Claude Codeの作者であるBoris Cherny、それから今は退職したxAIの創業メンバーIgor。スター選手たちがみんな話しているような構図でした。Andrejがこう言って、Borisも同じ感覚だと言って、要するに「カチッとはまる」みたいな話や大量のPRが起きている、という流れです。

<span class="paragraph-timestamp" data-ts="03:17">03:17</span> **ノ・ジョンソク** もちろんですね。

## Claudeの「Skills」機能とその仕組み    *08:58*

<span class="paragraph-timestamp" data-ts="08:58">08:58</span> **ノ・ジョンソク** それと、Claude Code SDKは何をつないでも動く、という体験がかなりできます。コーディングでなくても、会社の業務や単一のdatabaseにClaude Code SDKをつなぐだけで、実際かなりうまく動きます。

最近はバイオテックやresearch関連のskill packageもどんどん出てきています。そう、このHarnessは実際に新しいsource codeになってきています。

<span class="paragraph-timestamp" data-ts="09:27">09:27</span> **チェ・スンジュン** はい、Claude Code自体も…これは公式アカウントではなくchangelog共有アカウントですが、今出たところなんです。2.1が上がったと思ったら、12時間前に2.1.2が上がっていました。2.1で大きなversion bumpがあったんですが、日が経つごとにこうして継続的に上がっていくのを見ると、これは人間だけではできないですよね。Claude Codeは公式にClaudeが作っていて、Claude CodeがClaude Codeを作っているので、機能が非常に速く追加される切迫した状況です。

僕は年末年始に休みながら、基礎に戻りたかったのか、少しゆっくり考えていました。でもやっぱりClaudeのskillsはすごいと思います。年末にそのskillsのGitHubドキュメントを丁寧に読み込みました。かなりしっかり読みました。

以前に翻訳して読み込んだskillsで、これがそのrepoです。ここでskillsに入ると、例えば僕がよく使うalgorithmic art skillは、まず哲学を考え、コードに入る前に何をすべきかが書かれていて、翻訳して読むととても面白いです。実際に内蔵されているskillsとは少し違うかもしれないと言われていますが、GitHubで公開されたものにも読む価値のある点が多くて、楽しく読んでいました。

その中にSkill Creatorというskillがあります。つまりskillを作るためのskillが入っていて、今日はそれについて少し話したいです。

<span class="paragraph-timestamp" data-ts="11:16">11:16</span> **ノ・ジョンソク** skillは能力のmarkdown記述、説明promptの塊に、関連コードや例として使えるものをこのfolder構造でまとめたもので、これをskillと呼んでいます。

<span class="paragraph-timestamp" data-ts="11:33">11:33</span> **チェ・スンジュン** その通りです。OpenAIも昨年末から使い始めました。今やskillsはmarketplaceのようになっていて、すでに数千単位で共有されていると思います。そして簡単に作れるのが大きい。Claude Codeではもちろん、web interfaceでも会話している途中にアイデアが出ると作れます。このアイデアの背景はあとで少し話しますが、コンテキストを進めると会話の重力ができますよね。今使っている用語、モデルが返してきた用語、その状況で分布を少し広げて別の空間を探りたいとき、別の用語が必要になるのに、それが出にくいことがあります。

そこで使えるのが、ランダムなアルファベット4文字を1セットとして100セット生成して、それをacronymとして読ませる方法です。完全一致でなくてもよくて、連想能力、モデルの連想能力を引き出す狙いです。pareidoliaという概念があります。人が雲を見て動物や顔の形を見出す現象です。人間には連想する力があり、今のモデルにも強い連想能力があります。

ただし、その連想能力を発火させるには適切なtokenが必要です。だからこれらのacronymを読ませると、このPython codeが生成されます。実際にランダムな4文字組み合わせを100個ほど作って、そのうち10個ほどを現在の会話コンテキストに関連するacronymとして読ませます。たとえば「Self-understanding is temporary」が「Your Understanding of Self is Temporary」のように、儚さを帯びた面白いテキストが出てきます。僕にはできなくてもモデルにはできる。なら面白い分岐になるかもしれない。こうしてtokenが使われ、それに関連する会話ができるので、少し発散的・横断的な探索をするときに試しています。

これを長くやり取りした後、Skill Creatorをオンにしていました。このskillのためにSkill Creatorをオンにしていたんです。あれこれオンにしすぎると散らかるので、使いたいものだけオンにします。普段はオンにしていて、その状態で会話するとskillを作ろうとします。先ほどの4文字を作ってacronymで読む会話の流れでも、モデルが先にskillを提案してきました。「作る？」と言われて「作ろう」と言うと、folder構造を作ってこれが出てきます。各MD fileができたあと、クリックすると一括でmy skillsにコピーされます。ここが非常にスムーズです。

Claude、つまり…先ほど図で紹介したように、僕が以前から言っているのは「有用な副産物を生み出すpipelineを生み出すpipeline」という考え方です。会話をするとcode、再利用可能なテキスト、画像などの副産物が生まれます。そのイメージで何かを追っていくと、常に良い副産物が得られます。

そしてClaudeはそれを完全に製品化したという感覚がありました。ここでさらに一度改訂するプロセスがあって、全部終えた後my skillsにコピーしました。例えば「Domain Priming」。ここでは特定ドメインの用語を入れても入れなくてもよい。そうするとその方向のテキストが生成され、それを読みにいく行為自体がskillになります。ここを見るとcodeも入っていて、使い方の例もあるので、Claudeが必要に応じて使えるようになります。my skillsにあるのでオンオフもできます。

<span class="paragraph-timestamp" data-ts="16:01">16:01</span> **ノ・ジョンソク** はい、skill作成はかなり簡単です。だからClaudeでも使えるし、Claude Codeでも同じ構造でskillsを作ってどんどん拡張できます。

<span class="paragraph-timestamp" data-ts="16:13">16:13</span> **チェ・スンジュン** はい、なのでfolderをそのまま入れればいいだけです。

## なぜskillsはうまく効くのか？: 正確なtokenの重要性    *16:17*

<span class="paragraph-timestamp" data-ts="18:30">18:30</span> **ノ・ジョンソク** 「適切な」という表現、accurateとかappropriateとか…これは具体的に何を意味しますか？
