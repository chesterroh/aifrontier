## 开场：AI 点击时代的焦虑    *00:00*

<span class="paragraph-timestamp" data-ts="00:00">00:00</span> **卢正锡** 今天，也就是我们录制的这天，是 2026 年 2 月 21 日，星期六。上周我们和申正奎 CEO 做了那期节目，真的感觉开发节奏还在继续加速。

到处都是 click click click，一天做完的软件、一周做完的软件，高质量产品正在不断涌出，连我们自己都快跟不上了。与此同时，也开始出现一种情绪: 太挫败了，我都抑郁了，这到底该怎么应对，类似这样的观点还不少。

<span class="paragraph-timestamp" data-ts="00:33">00:33</span> **崔升准** 我不太会直接用“挫败”或“抑郁”这种词，至少不会明说，但我觉得内心深处确实有点这种感受。我觉得是从 OpenClaw 左右开始的。agent swarm、agent team 出来之后，那种一下子要同时推进很多事的趋势，让我感觉自己很难跟上，所以 1 月和 2 月我反而没有猛冲，而是放慢节奏，更像是在认真思考的模式里。但问题是，我慢下来、认真想的时候，背景却一直在飞速前进，于是又陷入了 FOMO 的恶性循环。

所以我想回顾一下最近发生的事，虽然我现在也没有所有答案，但想聊聊我自己是怎么在消化这件事的。

## MVK：在未知中朝正确方向前进    *01:15*

<span class="paragraph-timestamp" data-ts="01:15">01:15</span> **崔升准** 我 2 月初在 Uneven Future 做分享时，最后一页写的是:“即使不知道全部，也要朝正确方向前进，我们最少需要知道什么？” 这就是 MVK，Minimum Viable Knowledge。我们频道里也提过几次，我当时就是用这个框架收尾的。这个问题一直占据着我的脑子。到底该怎么真正弄清楚这个问题，我最近一直在和它缠斗，今天这场对话我觉得也会连到这条线。

<span class="paragraph-timestamp" data-ts="02:25">02:25</span> **卢正锡** 直接冲上去了。

<span class="paragraph-timestamp" data-ts="02:27">02:27</span> **崔升准** 对。而且 METR 说他们的任务集现在看起来开始饱和了，正在准备新的任务。所以这个图好像已经走到意义在减弱的阶段了。

然后今天早上我们一开始就收到“benchmark 饱和”的消息，就在今天早上。以及模型版本每次加 0.1 的趋势，似乎正在变成常态。我们本来期待 Sonnet 5，结果来了个 4.6。上周到现在我都还没认真试完。我用 Claude Opus 4.6 的习惯太深了，还是会一直开 Opus，不过我相信 4.6 也会很强。可能还更便宜。

## Gemini 3.1 Pro 与无穷无尽的新模型    *03:40*

<span class="paragraph-timestamp" data-ts="03:40">03:40</span> **崔升准** 然后 Gemini 3.1 Pro 也发布了。看 benchmark 的话，天花板看起来依然很高。感觉还能比现在更好，而且也确实在持续变好，所以后面还会继续上升。现在 Gemini 3.1 Pro 到了这个水平，Jeff Dean 也发了内容。你现在就能在 Google AI Studio 里试，但我昨晚开的一次 run 跑了 10 小时什么都没生成。我特别期待它会产出什么，结果录制前它告诉我报错了。所以性能我还没法完整验证。

但模型就是不停在出。后面还会有更多。我们到底该怎么评估这些？到这个阶段，我都有点……麻了。

<span class="paragraph-timestamp" data-ts="04:28">04:28</span> **卢正锡** 这些模型确实是通用性的顶点。几乎什么都能做，而最通用的能力反而最具体，这句话是前几天上 Y Combinator YouTube 频道的 Boris Cherny 说的，他做了 Claude Code。

而这种强大的通用智能，现在也在压过其他所有领域。最近受冲击很大的一个领域似乎就是科学，你估计也会讲这个。它几乎无所不能，看着都让人疲惫。

<span class="paragraph-timestamp" data-ts="05:01">05:01</span> **崔升准** 当然，日常真用起来还是有卡顿和磕绊的部分。但方向本身很明确，过去做不到的事现在确实在变得可行。这是 2 月 18 日，也就几天前。Chris Lattner，做 LLVM 的那位，后来做 Swift，在 Apple、Tesla 待过，现在有 Modular。因为他本来就是编译器背景，对编译器非常懂。Claude Code 的 C 编译器出来那天，他发帖说这是他见过最好的编译器相关文档，给人印象就是文档质量异常高，并且说要深入看看。后来他把这次 deep dive 详细写在博客里，我也翻译了。

## Chris Lattner 对 Claude C 编译器的深度评测    *05:11*

<span class="paragraph-timestamp" data-ts="05:56">05:56</span> **崔升准** 开篇就说这是那种有教材级价值的实现。赞美不少，当然也指出了缺点。整体上它有真实的方向感，不是 slop，值得研究。并且因为有 commit history，去看这个过程本身也很有意义，大概是这个意思。但文中也指出了哪里出了问题，Claude C 编译器最显眼的问题就是那个错误本身，最大问题是最后一段把测试需要的东西硬编码了，这说明它很难泛化到测试集之外。这个不是“意外”，而是很有信息量。当前 AI 系统非常擅长组合已知技术并朝可测指标优化，但对于生产级系统需要的那种开放式泛化仍然吃力。并且如果这个观察继续追问，就会变成一个更深的问题: 这对 AI coding 本身意味着什么？这部分也讨论得很有意思。我觉得这一段尤其重要。

限制方面也是，去实现已知抽象，和发明新抽象，是两回事。它能消掉重复苦工，让你从更接近当前最佳实践的起点开始，这本身是有价值的里程碑，但同时也暴露边界: 它不会发明新抽象，这次实现里也看不到真正新东西。尽管如此，它是教材级实现，这点成立。结尾他说得很好: 这不是软件终结，也不是工程师终结。这种说法太 hype。更像是把门彻底打开，因为实现越容易，真正创新的空间反而越大。降低实现门槛不会削弱工程师的重要性，反而抬高了 vision、judgment 和 taste 的价值。越容易做出来，越难的是判断“什么值得做”。AI 在加速执行，但意义、方向、责任，本质上仍然属于人类。这段也很深刻。写代码从来不是目标，目标是做有意义的软件。未来属于那些拥抱新工具、质疑既有假设、并设计出帮助人们协作构建系统的团队。

<span class="paragraph-timestamp" data-ts="08:23">08:23</span> **卢正锡** 申正奎 CEO 上周来也说了同样的话。归根结底，软件逻辑才是交付物，代码只是实现逻辑的中间工具，而这个中间层正在消失。

## Andrej Karpathy 的九的进军：90% 与 99.99% 之间    *08:35*

<span class="paragraph-timestamp" data-ts="08:35">08:35</span> **卢正锡** Andrej Karpathy 以前演讲里提过“March of Nines”这个概念。他就是用的这个词，“March of Nines”。先做到 90%，再从 90% 到 99%，再到 99.9、99.99，每多一个 9 都需要同等量级的努力。

所以 Claude C 编译器可能自己做到 90%，拉一拉也许能到 99%，但在做 99.99% 的高手眼里，剩下那 0.99% 可能是巨大的鸿沟。这个鸿沟落在哪，最终就会回到升准刚才总结的那些很“人”的东西: taste、意志之类。所以我们前面还有很多事可做，大概是这样的结论。

<span class="paragraph-timestamp" data-ts="09:30">09:30</span> **崔升准** 所以不管怎样，它意义重大，但还没到真正创新的层级，这是 Lattner 的意思。即便如此，这个方向意味着什么，变成新常态后我们能做什么，这两点仍然值得认真看，我想再强调一次。

然后提到了“taste”这个词。我们也一直在反复讲 taste。最近我提过几次的人，Corca 现任 CTO 姜圭永，好像正在写《有品味的软件》这样一篇内容，看起来很值得读，大家可以看看。

## 万物皆可搜索问题：科学与药物发现    *10:12*

<span class="paragraph-timestamp" data-ts="10:12">10:12</span> **崔升准** 但说到必须要有真正创新的地方，比如科学和数学，不能只是重复已知问题，必须有突破。现在恰恰就在发生这些事。

<span class="paragraph-timestamp" data-ts="10:23">10:23</span> **卢正锡** 这类事一直在出现，我们不可能全讲完。非常简化地说，药物发现就是识别导致问题的蛋白质，理解这个蛋白如何表达某种机制，然后找到抑制或促进这个过程的物质，也就是所谓抗体，用化学方式干预这个机制。大多数药物都是这个原理。

也就是说，归根结底它变成理解蛋白结构，再去找能匹配它的另一个结构的游戏。所以这个领域显然也可以完全还原成计算，作为搜索问题来解，现在正在被证明这一点。

## OpenAI 理论物理案例研究与半人马时代    *11:07*

<span class="paragraph-timestamp" data-ts="11:07">11:07</span> **崔升准** 先放下这个不谈。就我自己翻译时的细读来说，我读得最仔细的是 OpenAI 在理论物理上的工作，一篇具体论文。OpenAI 与科学那篇内容，是我去年末重点讲过的，我介绍过 Kevin Weil 和研究黑洞的科学家那个案例，之前提过一次。

那位研究者把其他科学家，包括他自己的导师，也就是理论物理最前沿的人，拉进了 OpenAI，合作做了一些事后，论文就出来了。那位加入 OpenAI 的黑洞研究者的导师，哪怕一年前也不确定 AI 能帮多大忙，大概 2024 年，甚至 2025 年初还是这种心态。但一年后，在和 GPT-5.2 Pro 多轮交流之后，他向 OpenAI 内部模型发了最后一个 query，这个模型不仅解出了一个此前未解的量子场论问题，还给出了证明。

12 小时，故事是这样的。也就是说，在那个领域，模型完成了两个全球最聪明的人都做不到的事。我和他们在一起时，他们非常兴奋，大概是这样。

<span class="paragraph-timestamp" data-ts="12:29">12:29</span> **卢正锡** 对，没错。

<span class="paragraph-timestamp" data-ts="12:30">12:30</span> **崔升准** coding 其实已经过了半人马时代，现在感觉科学也在进入半人马时代。Gary Kasparov 输给 Deep Blue 之后，象棋界出现了 centaur 的概念。centaur 是半人半马。意思是 AI 和人类组队，会比纯 AI 或纯人类更强，那种 centaur chess 曾经流行过一阵。

现在科学终于也进入了和 AI 协同的半人马时代，而 coding 其实已经走得更远，把更多工作交给了 AI，现在科学和数学好像也在沿着这条轨迹走。

<span class="paragraph-timestamp" data-ts="13:11">13:11</span> **卢正锡** 真的感觉所有问题都在被重写成搜索问题。只要继续投入更多电力和计算，inference token 就在替我们探索未知。过去人类一步步思考推进的事情，现在都在被纯计算替代。

<span class="paragraph-timestamp" data-ts="13:33">13:33</span> **崔升准** 他们那段经历在 Twitter 上也一直来回争论。但不管怎样，那篇博客周围的人和研究者本人都在说: AI 在 2026 年对物理的作用，会类似它在 2025 年对 coding 的作用。他们是这么说的。我把翻译发到社交媒体后，韩国这边也有几位科学家留言，这点让我印象很深。像“这真的在发生，我得准备了，研究生阶段会变成什么样”之类的感想，确实出现了。

## 模型能力的本质：插值还是外推？    *14:14*

<span class="paragraph-timestamp" data-ts="14:14">14:14</span> **卢正锡** 对，这也是我们早期就聊过的。为什么讨论模型进步时，很多人一直说模型只是简单的插值引擎，这是早期很常见的看法。说它只会插值，无法进入真正新领域，当时模型还比较弱，大家是这么说的。但现在看结果，几乎反过来了，看起来都像在外推，超出已知范围。但我自己的理解是，如果把模型正在投影的某种宇宙真理总量记作 100，这个 100 在模型里是非常稀疏地被表示的。

我依然认为它是插值。只是在一个巨大空间里对这些稀疏点做插值，从人的视角看起来就像外推。

<span class="paragraph-timestamp" data-ts="15:13">15:13</span> **崔升准** 如果插值发生在极大的域里，它很可能看起来像新东西，事实上也可能真的会产生新东西。

<span class="paragraph-timestamp" data-ts="15:20">15:20</span> **卢正锡** 我们所谓“人类独有创造力”，其实只是有人先发现了一个本来就存在但尚未被发现的空间。如果把宇宙真理看作 100，大概就是这么回事。所以随着这些模型计算继续增长、通用性继续增强，中间那些东西都会变得可达，当然需要时间。但如果假设最终都会变得可行，那我们是不是就该基于这个前提去构建？我们在说的是模型有现实可能把一切做完。

<span class="paragraph-timestamp" data-ts="15:53">15:53</span> **崔升准** 但从前面那场展示看，给人的印象是没有革命性新东西，不过如果把那当作新常态，下一步该做什么？重点是它确实交付了非常巨大的成果。关于这个物理故事我一直在盯的一点是: 多轮交流后他们调用了内部模型，另一篇帖里把它称为内部模型，同时也提到 OpenAI 尚未公开的一种特殊 scaffolding 或 harness。GPT-5.2 Pro 单独做不到的事，在那个特殊 harness 里跑了 12 小时后做到了，故事是这样。我是这么理解的。

但除了 harness 本身，那些来回交流也很关键。为什么他们能做到？当然因为他们是顶级专家、世界级物理学家，所以能做到，但最终都收敛成 prompt。那在被压缩成 prompt 之前到底是什么？如果是我输入同样 prompt，我大概也看不懂答案，但 model 和 harness 会给同样输出，model 和 harness 会。所以这点一直让我在意。他们到底用了什么词、什么句子？

<span class="paragraph-timestamp" data-ts="17:13">17:13</span> **卢正锡** 这就连到了升准 1 月初说的那个点，好的 prompting，prompting 做对，直接相关。每走一步，模型内部空间都在变。它所表示的空间在每一步都在变化，表示空间持续重构，靠这个不断找到前路。所以持续让那个空间发生迁移的能力，也许就是我们说的 taste，或者定方向，某种人的价值。现在大家都在一种模糊、摸索的状态里感受到这个，对吧？

<span class="paragraph-timestamp" data-ts="17:48">17:48</span> **崔升准** 对。上周我和正奎聊时，他也让我们看到，他从去年中那种长 spec 写作风格，重新回到了现在这种高频来回模式。我们上期举的例子比较直白，但正奎肯定用了他自己的词汇和领域知识，这些都会体现在 token 里。

<span class="paragraph-timestamp" data-ts="18:17">18:17</span> **卢正锡** 当然。最近我跟很多做 hardcore agentic coding 的人交流，从这些对话看，过去那种像 oh-my-opencode 的方式，死命朝目标冲、要求不管怎样都必须完成、接上就一直跑下去，用这种方式，正如我们前面 C 编译器例子说的，最多也就是停在已知范围内。大家开始有这种认知了。所以要进入那种拓展地平线的工作，人类在环 absolutely 必要。理解上下文并持续输入 prompt，把方向不断往不同侧面引导，这就完全违反 ralph loop。

## Agentic Coding 的现实：Ralph Loop 的极限    *18:19*

## 99 与 99.99 的差异    *46:23*

<span class="paragraph-timestamp" data-ts="46:30">46:30</span> **卢正锡** 因为我是做生意的人，我总会把这件事翻译成商业视角，花很多时间做思想实验: 里面有哪些商业机会。这样做着，你会一路遇到有趣的人，也会看到“哦这个人在试这种路径，那个人在试那种路径”。我从中非常明确地感受到一点: 大多数人只想要答案。但 AI 领域复杂性纠缠太深，不能拿一句话答案就直接跑。它有太多层交织在一起。

从层次看，现在模型被像库一样封装起来，打包、下沉，变成你调用就给答案的对象。但你要在这上面做出真正业务，不看模型内部，你就很难判断这个行业会往哪走。你看不见，这就是差异。以前大家说会 Python 就不必懂底层 C 或汇编。但 AI 不一样，你得从硬件计算层一路上到模型，再从模型一路上到业务，你才开始看到答案。

上周正奎提过这个。他说他又开始“雕模型”了。他说所有软件、每个工作单元，最终都会进到一个单模型里，上面只覆盖一层薄 harness，他是这样设想结构的。所以现在我们都在 frontier model 之上发 API call 构建应用。但正奎像是得出了一个结论: 即便如此，细到每个小细节，模型时代会回来。我是这么理解他的。到那时，小模型会成为新的 web service。

那要做到这点，难道只要拿一个现成 B2B SaaS，塞一本到你熟悉的书，再 fine-tuning 一下就够了吗？还是说，能在这个空间里更深地应用 cutting edge 的人会有优势？如果所有人都在 99、99.9、99.99 之间竞争，客户会全部流向 99.99，不会去 99。这意味着我们过去的速度感、价值感都在变化。一旦重新进入这个空间，内部的相对方差会再次成为新常态并继续拉大。

这件事发生在我们这一代，是不幸；但反过来看，对 startup，或者对那些迫切需要变化的个人，这全是机会。别人的危机就是我们的机会。我们必须这么重构认知。
