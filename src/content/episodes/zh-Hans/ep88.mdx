## GLM 5 报告与以 RL 为中心的技术创新    *02:23*

<span class="paragraph-timestamp" data-ts="02:25">02:25</span> **金成贤** 最近虽然不是 DeepSeek V4，但有一份非常有意思的前沿级研究报告出来了，就是 GLM 5。这是 Zhipu AI 的报告，里面有很多很有意思的技术创新和细节。非常粗略地总结有三点。第一是把 RL 做得更高效，比如 Sparse Attention，以及为 RL 搭基础设施，像 async RL，这类大量技术细节里，很大一部分都在做 RL 的效率优化、算力效率优化，也包括更高效地利用 context length 来推进 RL。第二是对 RL 的一些 objective 做微调，让它更稳定。第三是让 RL 更多样化。最后我可能会讲到环境扩展，也就是让 RL 能覆盖更多样的任务，并且更高效地扩展环境。这些是主要主题。

在 GLM 5 这份报告里，大量创新几乎都和 RL 有关，方向就是把 RL 做得更高效、更稳定、更多样。想到这里我就会想到 Yao Shunyu 的 “The Second Half”。他说 “The First Half” 是一个“有 benchmark，就围绕 benchmark 开发方法；方法把 benchmark 解开后，再做新 benchmark”的时代；而到了 “The Second Half”，我们已经找到了方法上的答案，拿着这份答案去套各种问题就可以了。他这个表述某种意义上看起来挺“傲慢”的，但同时我觉得它确实准确，至少从 2025 年到现在这一年里非常准确。大家都把 RL 当成了答案，几乎所有努力都集中在“如何把 RL 做得更好”以及“如何扩展 RL”，现在更是如此。

## Yao Shunyu 的 The Second Half：RL 这份答案    *03:40*

<span class="paragraph-timestamp" data-ts="04:37">04:37</span> **金成贤** 一提到“技术”或者“技术创新”，大家感受到的往往是“方法创新”，但在方法创新上能讲的、有趣的东西其实不多，因为一直都是 RL。我们只能反复讲 RL 如何继续扩展、如何变得更深，而不是讲某条完全不同的技术轨迹。所以我才会觉得，用一条技术轨道去讲“它现在怎么发展、未来会怎么发展”有点困难。因为至少可预期的是 RL 会持续被应用，并且持续渐进式发展，而这正是当下正在发生的事。在这个层面上，我也会想到“秘密配方”。如果说某些前沿模型有秘密配方，我们通常会觉得那更接近方法层面的创新，好像他们掌握了某种完全未知的秘密，不知道这个秘密就做不出模型，人们就容易相信确实存在这种东西。

## 秘密配方真的存在吗    *05:16*

<span class="paragraph-timestamp" data-ts="05:41">05:41</span> **金成贤** 当然，创新非常多。像 GLM 5 那样，有非常多创新，也有非常多改进。我并不是在否定这些改进，MoE 之类也都是极其重要的创新。但如果从 GPT-3.5 到现在一路看下来，这些更像是随着模型持续渐进式改进和开发，自然会出现、也能被理解的东西。真正最接近“秘密配方”的可能是 RLVR。除此之外，多数虽然是重要创新，但并不是那种“你不知道就绝对追不上、而且极难知道”的秘密配方。当然我有这种感受，可能也因为中国公司把很多秘密配方都自己找出来或开发出来并公开了，所以它们看起来不再神秘，但我现在的体感就是这样。这不只是 Agent RL 或 RL 本身带来的，而是到目前为止很多创新总体都处在一种渐进且可预期的轨迹里。

从这个角度看，现在更重要也更显而易见的是：当下比起“惊人的新创新、全新的方法”，当然我不是否认小的创新，它们会持续出现、以后也会继续出现，但更关键的是扎实做基本功。做出更好的数据、搭建稳定基础设施、投入大量算力去做模型，这些最基础的东西。不是某种创造性奇招，而是这些基础正在主导当前模型性能。不同模型有不同性能差异，每个人也有偏好模型，但这些偏好背后，与其说是某家公司掌握了别人完全不知道的配方，不如说更接近基本功问题：更好的数据、更好的 scaling。这类问题才是核心。我认为现在关键就是把基本功做扎实。也因此，只要能把基本功做扎实，很多后发者是有机会追上的，环境似乎也具备了。与此同时，前沿公司因为在这些基本功上积累了大量经验，确实仍处在更有时间优势的位置。

## 基本功时代：数据与产品感    *07:14*

<span class="paragraph-timestamp" data-ts="08:18">08:18</span> **金成贤** 同时我还在想一点：所谓“扎实做基本功”，其实和做产品、对待产品的态度高度相关。我这话可能也受限于我接触过的 AI 组织，但我认为 AI 组织通常研究导向会更强。工程师的关注会更偏向研究创新、全新的研究方法。但我觉得做产品需要另一种感知。经历大量迭代、大量试错、大量真实使用，然后一点一点改进、持续打磨模型，这个过程非常重要，而这种感知和态度的重要性现在在显著上升。你一点点打磨后产生的差异，对用户可能会非常明显；反之，人就更容易偏向那些看起来能快速大幅拉升性能数字的东西，而不是持续补齐边角、持续打磨。但当下比那更重要的，是把产品做出来、把基本功做好。因为很多关键方法其实已经被发现了。某种意义上正因为 RL 这个方法已经被找到，所以在这个方法内部持续打磨，才会拉开很关键的差距。

<span class="paragraph-timestamp" data-ts="10:00">10:00</span> **卢正锡** 我同意。确实过去两个月，或者说从去年秋天到现在成贤你刚讲的这个时间点，模型并不是突然激进地一下好 2 倍、3 倍。它一直在变好，但实际上大家好像都在判断：现在的模型已经足够可用了，非常非常不错。所以围绕它也出现了很多产品。

## 环境扩展：Agent RL 的瓶颈    *15:15*

<span class="paragraph-timestamp" data-ts="15:15">15:15</span> **金成贤** 虽然预测很难，但我觉得相对更容易估量的问题之一是环境扩展。这里说的是 RL 里的环境。所谓 RL 里的环境，就是某个 agent 进入其中执行 action、开展活动，观察环境变化，再基于观察最终获得 reward，这样一套发生机制所对应的环境。

现在说到 agent，LLM agent，比如软件工程任务就可以是一个环境。你让它修这个 bug，它面前有 bug、有源码、有可用工具，然后通过这些工具交互，最终拿到某种 reward。未来必须不断增加这类环境的多样性。因为现在任务从简单走向更多样、更复杂。AI agent 现在能做的事情已经很多了，而且能做更复杂的程序。那就要构建能支持更复杂程序的环境。以前可能只是做一个函数，之后会变成做一个完整程序，再往后可能是整套服务。这样环境必须不断扩展，而扩展带来的问题是：随着能力层级提高，要构建的环境复杂性也会持续抬升。

![环境扩展 — 4种技术发展情景（1: Bounded, 2: Linear, 3: Exponential, 4: Innovation waves）](/episodes/ep88/env_scaling_graph.jpg)

<span class="paragraph-timestamp" data-ts="16:40">16:40</span> **崔升准** 这张图你可以给大家读一下。

<span class="paragraph-timestamp" data-ts="16:44">16:44</span> **金成贤** 那我们就假设在这个局面里，“环境扩展”是最大瓶颈，并且“环境多样化”是模型发展中最关键关口。沿用 The Second Half 的前提：RL 是答案，只要有可做 RL 的环境就什么问题都能解。若如此，既然问题都能解，反而“构建环境”会成为唯一技术瓶颈。那我们就想想这个瓶颈可能会沿什么轨迹被解决。

最乐观的人可能会认为，时间越往后这个技术瓶颈会越容易被解决。这是可能的。比如在某个中间节点突然找到一种方法，让环境扩展、构造多样任务一下子变得非常容易，那曲线就会指数级猛升。这个可以发生的

<span class="paragraph-timestamp" data-ts="17:40">17:40</span> **崔升准** 可能性就出现了。

<span class="paragraph-timestamp" data-ts="17:42">17:42</span> **金成贤** 这就是绿色曲线。它会持续拉升。我觉得一个可能案例是 Continual Learning 问题被解决时，就可能出现类似情况。那环境扩展问题就会被技术性地解决，接近一种决定性解决。那就会指数级发展。我认为持续性发展是可能的，当然这种情景下也不是完全没有现实约束。

悲观地看，因为复杂性持续上升，构建环境的成本和时间也会持续上升，一旦复杂性达到不可承受水平，发展就会停滞。那就会像蓝色曲线那样趋近。

我觉得这更接近“几乎没有技术创新”的情形。几乎没有创新，只能像现在这样持续手工构建，再加上复杂性持续升高，手工成本也按复杂性同比增加，就会更接近蓝色。反过来，如果有一些技术创新和改进，并且构建新环境、更复杂环境的成本不是无止境持续上升，那就会更接近线性，持续渐进上升。

即便是渐进上升，也不代表体感一定是渐进的。技术进步的差异可能很渐进，但人的体感可以非常剧烈。不过只看技术本身，可能就是这种渐进形态。

再细看这个“渐进”，我觉得更像这样：先一点点推进，某次创新拉一段，然后复杂性变大速度又慢下来；接着又靠某些技术发展或创新把复杂性再降下来。从远距离看就会近似一条直线。
