更像是如果你从 GPT-3.5 一路看到现在，

这些都属于在模型持续改进、发展过程中

秘密配方。

## Environment scaling：Agentic RL 的瓶颈    *15:15*

<span class="paragraph-timestamp" data-ts="15:15">15:15</span> **金成贤** 我觉得相对容易判断的问题之一是

environment scaling 问题。

environment scaling，我说的是 RL 里的环境。

在 RL 里说环境，

就是 agent 进入环境、执行动作、

在其中操作、观察环境变化，

最后基于这些观察获得 reward。

我们说的是这种环境。

对当前 agent 来说，也就是 LLM agent，

比如软件工程任务，

就是这样一种环境。

“修复这个 bug”，这里有 bug，

这里有源代码，在这份源代码里

你可以使用各种工具，和这些工具交互后，

最终拿到某种 reward。

这些环境之后需要越来越多样化。

因为现在我们做的是简单任务，

但接下来会越来越多地处理更丰富、更复杂的任务。

AI agent 今天已经能做很多事了。

它们也越来越能构建复杂程序。

所以我们需要构建环境，

让它们去构建越来越复杂的程序。

以前可能只是写一个简单函数，

接下来是构建整个程序，

再往后是从零搭建完整服务。

这些环境需要不断扩展，

但随扩展带来的问题是，

门槛持续上升时，

我们要构建的环境复杂度

也会同步提高。

我认为是最关键的关卡。

如果我们接受《下半场》并去做 RL，

就假设任何问题都能解，

那么既然问题都可解，
